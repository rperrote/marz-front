# i18n

Lingui para internacionalización. Cargar cuando agregues copy, modifiques mensajes user-facing, o trabajes con catálogos.

## Setup

- `src/shared/i18n/setup.ts` — configura el activador.
- `src/shared/i18n/provider.tsx` — `<I18nProvider>` envuelve la app en `__root.tsx`.

## Macros

Usar siempre macros (`@lingui/core/macro`, `@lingui/react/macro`). Compilan en build.

```tsx
import { t } from '@lingui/core/macro'
import { Trans, Plural } from '@lingui/react/macro'

// String simple
const label = t`Continuar`

// Con variables
const greeting = t`Hola, ${firstName}`

// JSX con markup
<Trans>Bienvenido, <strong>{name}</strong></Trans>

// Plurales
<Plural value={count} one="# campaña" other="# campañas" />
```

## Reglas

- **Todo string user-facing** pasa por `t\`\``o`<Trans>`. Hardcodear strings en JSX es un bug.
- **Identifiers de código en inglés**. Solo el copy va en español.
- **Comentarios en inglés** (cuando los hay).
- **Strings dinámicos** (ej. `t\`${count} de 5 seleccionados\``) sí se aceptan — Lingui los maneja.
- **Strings de error técnicos** (ej. internal logging) NO necesitan i18n. Solo lo que ve el usuario.

## Catálogos

Los catálogos compilados viven en `src/shared/i18n/locales/`. Se generan con el CLI de Lingui (chequear `package.json` para el comando exacto). Si agregás un mensaje nuevo, regenerar y traducir antes de mergear.

## Convención de copy

Español argentino, voseo natural. Estilo:

- Directo, sin relleno.
- Sin emojis.
- Sin "querido usuario", sin disculpas.
- Mensajes de error: qué pasa + qué hacer. No solo "Error".

Ejemplos:

- "Continuar" no "Hacer click para continuar".
- "Algo salió mal. Intentá de nuevo." no "Lo sentimos, ha ocurrido un error".
- "Sin take rate" no "0% de comisión".
